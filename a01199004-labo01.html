¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet considero que es la parte donde todos están conectados y la WWW es un componente dentro de esta misma que permite visulizar información por un protocolo llamado HTTP.

¿Cuáles son las partes de un URL?

Contiene el protocolo (HTTP/HTTPS), luego el nombre de dominio (www.google.com) o una direccion ip (8.8.8.8), a veces se incluye el puerto (:80), y sigue de una ruta a la consulta específica (courses/5209).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

-GET: Recupera datos.

-HEAD: Recupera casi igual que GET, lo que la hace diferente es que no lleva el cuerpo.

-POST: Lo que hace es enviar una entidad a un recurso en específico, esto causa un cambio en el servidor.

-PUT: El método put reemplaza las propiedades del recurso local y elimina aquellos que no están incluidos en la solicitud.

-PATCH: Este método se utiliza para aplicar modificaciones parciales a un recurso en específico.

-DELETE: Este método borra un recurso específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método POST es el que se debe utilizar, ya que este se encarga de enviar datos al servidor, por ejemplo cuando envias datos de contacto.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método utilizado es GET ya que este es el que recupera información de un servidor.

¿Un servidor web devuelve una respuesta HTTP con código 200, ¿Qué significa esto? ¿Ocurrió algún error?

No ocurrió ningún error, al contrario este código se usa frecuentemente para indicar que se recibió, procesó una solicitud y que esta se completó exitosamente.

¿Es responsabilidad del desarrollador corregir un stiio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

La responsabilidad es algo subjetivo para mi en este apartado, por ejemplo si el desarrollador cometió un error a la hora de crear la página y darle sus adecuados redireccionamientos, si tiene responsabilidad; en cambio, si el usuario que interactua con el sitio introdujo mal la URL no es culpa del desarrollador. Hay maneras de resolverlo, dependiendo del contexto, además es recomendable tener una página de error 404 personalizada que ayude al usuario a navegar por la página y evitar que este la abandone.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Aquí en este caso si es responsabilidad del desarrollador porque quiere decir que la página encontró un error interno en el servidor, comúnmente por errores de programación..

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

-La compatibilidad entre navegadores.

-El soporte multimedia sin dependencias.

-Mejor rendimiento y soporte móvil.

¿Qué componentes de estructura y estilo tiene una tabla?

Contienen celdas organizadas en filas y columnas, puede llevar una leyenda y pueden hacer referencia a columnas

-table para crear la tabla.

-td o th para un conjunto de celdas.

-tr para organizarlos en filas que se pueden organizar en grupos de filas (thead, tbody, tfoot).

-caption para ponerle una leyenda.

-col o colspan para hacer referencias a columnas.
