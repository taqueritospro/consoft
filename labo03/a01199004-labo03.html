¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet considero que es la parte donde todos están conectados y la WWW es un componente dentro de esta misma que permite visualizar información por un protocolo llamado HTTP.

¿Cuáles son las partes de un URL?

Contiene el protocolo (HTTP/HTTPS), luego el nombre de dominio (www.google.com) o una dirección IP (8.8.8.8), a veces se incluye el puerto (:80), y sigue una ruta a la consulta específica (courses/5209).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

- GET: Recupera datos.

- HEAD: Recupera casi igual que GET, pero sin el cuerpo de la respuesta.

- POST: Envía datos al servidor.

- PUT: Reemplaza todo un recurso.

- PATCH: Modifica parcialmente un recurso.

- DELETE: Borra un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

El método POST, ya que envía datos sensibles que no deben verse en la URL.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método GET.

¿Un servidor web devuelve una respuesta HTTP con código 200, ¿Qué significa esto? ¿Ocurrió algún error?

Significa que todo está bien; no ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende: si el error es por mala programación (enlaces rotos), sí es su responsabilidad; si el usuario escribió mal la URL, no es culpa del desarrollador, aunque siempre es recomendable tener una página 404 personalizada.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, porque el error 500 indica un problema interno en el servidor, causado casi siempre por errores de programación.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)?

Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que ese atributo ya no se recomienda usar y no será soportado en nuevas versiones. Algunos ejemplos son: acronym, big, center, dir, font.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

- Mejor soporte multimedia nativo.

- Mejor soporte móvil.

- Compatibilidad mejorada entre navegadores.

¿Qué componentes de estructura y estilo tiene una tabla?

- table, tr, td, th, caption, thead, tbody, tfoot, col, colspan.

¿Cuáles son los principales controles de una forma HTML5?

- email, url, date, number, range, color, entre otros.

Preguntas LABO03

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Sirve solo cuando se trabaja con frameworks de estilo para anular cualquier cosa que venga con el framework, pero no se usa mucho porque las prioridades de manera natural están organizadas.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Puede afectar la legibilidad, la accesibilidad y el rendimiento de carga del sitio; por consecuencia puede afectar las visitas de los usuarios en la página.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?

Usar % para diseños flexibles y que no requieran algo fijo, px para hacerlo con precisión para ajustarlo en la pantalla y evitar pt porque es para impresión.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Se reduce el tiempo de carga del archivo ya que disminuye su tamaño.
