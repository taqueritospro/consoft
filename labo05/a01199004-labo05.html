¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet considero que es la parte donde todos están conectados y la WWW es un componente dentro de esta misma que permite visualizar información por un protocolo llamado HTTP.

¿Cuáles son las partes de un URL?

Contiene el protocolo (HTTP/HTTPS), luego el nombre de dominio (www.google.com) o una dirección IP (8.8.8.8), a veces se incluye el puerto (:80), y sigue una ruta a la consulta específica (courses/5209).

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Recupera datos.
  • HEAD: Igual que GET, pero sin cuerpo.
  • POST: Envía una entidad al servidor.
  • PUT: Reemplaza un recurso por completo.
  • PATCH: Modifica parcialmente un recurso.
  • DELETE: Elimina un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML?

El método POST, ya que envía datos al servidor.

¿Qué método HTTP se utiliza al acceder a una página desde el navegador?

Se utiliza el método GET.

¿Qué significa un código 200?

Significa que la solicitud fue exitosa, no hubo error.

¿Es responsabilidad del desarrollador ante un error 404?

Depende: si fue por mala configuración del desarrollador, sí; si fue por error del usuario, no. Aun así, es buena práctica tener una página 404 personalizada.

¿Es responsabilidad del desarrollador ante un error 500?

Sí, indica un problema en el servidor usualmente por errores de programación.

¿Qué significa que un atributo esté desaprobado en HTML5?

Que ya no es compatible o válido. Ejemplos: acronym, big, center, dir, font.

Diferencias principales entre HTML4 y HTML5

  • Soporte multimedia nativo
